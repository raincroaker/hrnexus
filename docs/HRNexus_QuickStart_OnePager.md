# HRNexus Quick Start (One-Page Handover)

## Access
- **URL:** [Your Hosted HRNexus URL]
- **Default Admin (seeded):**
  - Email: `admin@hrnexus.com`
  - Password: `password`
- **Important:** Change the default password immediately after first login.

## First Login Checklist (Admin)
1. Go to `Settings` -> change password.
2. Verify `Dashboard` loads correctly.
3. Confirm these modules are visible: `Employees`, `Attendance`, `Chats`, `Calendar`, `Documents`, `Settings`.

## Go-Live Setup Flow
1. **Employees**
   - Create departments and positions.
   - Add initial employees and assign roles (admin/manager/employee).
2. **Attendance**
   - Configure time in/out and break settings.
   - Validate leave and overtime workflows.
3. **Documents**
   - Upload sample file.
   - Test approve/reject, preview, and search.
4. **Chats**
   - Create a chat group, add members, and send test messages.
5. **Calendar**
   - Create one event category and one sample event.

## Role Snapshot
- **Admin:** Full access to all modules and setup actions.
- **Department Manager:** Team-level operational actions.
- **Employee:** Personal attendance/profile and assigned module access.

## Daily Operations (At a Glance)
- **HR/Admin:** attendance review, leave/overtime updates, employee maintenance.
- **Managers:** team attendance tracking, coordination in chats.
- **Employees:** check attendance records, view documents, update own account settings.

## Common Issues
- **Cannot login:** Verify credentials, use reset flow (if enabled), ask admin to validate account status.
- **Missing menu/module:** Role-based access may restrict visibility.
- **Data not updated:** Refresh page; verify save confirmation toast/message.
- **Document not previewing/searching:** Check approval/access status first.

## Stakeholder Acceptance Checks
- Default admin password changed.
- Core employees/departments configured.
- Attendance settings validated.
- One end-to-end test completed per module.
- Pilot users confirmed successful daily usage.
